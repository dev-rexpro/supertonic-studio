# Supertonic 3 — Voice Clone Studio

[![Open In Colab](https://colab.research.google.com/assets/colab-badge.svg)](supertonic_studio.ipynb)

A comprehensive Web GUI Studio and script interface to optimize and extract voice styles (`.json` files) for the Supertone/supertonic-3 model.

![project-logo](assets/cloner-logo.jpg)

---

## ✨ Features
* **🎙 Web GUI Studio**: A premium, responsive single-page web interface with a dark theme.
* **🌐 Local & Cloud Connections**: Toggle between a local backend or remote server (e.g. running on Google Colab) with connection testing.
* **⚡ Live Progress Logs**: Stream training progress step-by-step (`Step X/3000 | Loss: ... | LR: ...`) directly in the UI console.
* **📁 Audio Management**: Upload target voice files, play previews, synthesize speech, and play/download generated samples.
* **☁️ GPU-Accelerated Colab**: Start a Google Colab GPU session using the included notebook.

---

## 🚀 Running the Web GUI Studio

### Option A: Local Run (Your Machine)
1. **Install Dependencies**:
   ```bash
   pip install -r requirements.txt
   pip install fastapi uvicorn python-multipart
   ```
2. **Start the Server**:
   ```bash
   python server.py
   ```
3. **Open in Browser**:
   Go to [http://localhost:8000](http://localhost:8000)

---

### Option B: Cloud Run (Google Colab with GPU T4)
For fast GPU-accelerated style extraction without local GPU requirements:
1. Open the notebook [supertonic_studio.ipynb](supertonic_studio.ipynb) in Google Colab.
2. Ensure the Colab Runtime is set to **T4 GPU** (*Runtime -> Change runtime type -> T4 GPU*).
3. Run the cells. The notebook will automatically clone the repository from GitHub.
4. Under Section 4, a public **Ngrok** URL will appear (e.g. `https://xxxx.ngrok-free.app`).
5. Open that link to launch the Studio. You can upload voices, run training, and generate audio on Colab's GPU.
6. Once training completes, download the generated style `.json` file directly from the Colab file manager pane at:
   `/content/supertonic-studio/logs/<voice_name>/<voice_name>.json` or download it using the **Download Style** button in the Web UI.

## ▶️ Script Usage (CLI alternative)

Example training run via command line:
```bash
python train_style.py \
  --name my-voice \
  --target-wav-path voices/my-voice.wav \
  --num-steps 3000 \
  --learning-rate 0.0002
```

Arguments reference:
* `--name`: Output name for logs and style JSON file.
* `--target-wav-path`: Path to the source WAV file.
* `--reference-style`: Choose `auto` (default) to match best built-in style, `none` for random init, or path to a specific JSON style.
* `--seed`, `--speed`, `--num-steps`, `--learning-rate`, `--vocoder-steps`: Training knobs affecting quality and speed.

## Limitations

While this approach captures speaker identity, its ability to clone emotional expressiveness is limited. This is because the current loss function prioritizes identity over prosody. Furthermore, the original speaker encoder was not released by the developers.
Check the `samples/` folder for examples of the current results.
