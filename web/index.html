Complete guide to using Supertonic3 Voice Clone Studio

🚀 Getting Started

Supertonic3 Voice Clone Studio lets you clone any voice through a simple three-step workflow:

  1. Upload — Add a target voice audio file (WAV recommended, 10–60 seconds of clean speech)
  2. Train — Run voice style optimization to learn the speaker's characteristics
  3. Generate — Synthesize new speech with the cloned voice using any text
💡 Tip: For best results, use a clean recording with minimal background noise. Longer recordings (30–60s) generally produce better voice clones.

🎙 Voices

The Voices section is where you upload and manage the target audio files used for training.

How to upload:

  • Drag & drop audio files onto the upload zone, or click to browse
  • Supported formats: .wav, .mp3, .flac, .ogg
  • You can upload multiple files at once

Managing voices:

  • Use the built-in audio player to preview each uploaded file
  • Click the trash icon to delete a voice file
  • Voice files are stored in the voices/ directory
ℹ️ Recommended: Use mono WAV files at 44.1 kHz sample rate for optimal quality. Keep recordings between 10–60 seconds of continuous speech.

⚡ Training

The Training section configures and runs voice style optimization. This process learns a style vector that captures the speaker's voice characteristics.

Workflow:

  1. Select a Target Voice from your uploaded files
  2. Set Gender to match the target speaker (important for initial reference style)
  3. Adjust training parameters as needed (defaults work well for most cases)
  4. Click Start Training and monitor progress in the log panel
  5. Training saves checkpoints every N steps and a final style when complete

Parameters reference:

Parameter Default Description
Name mb4 Output name for logs and saved style files
Gender Male Voice gender — selects the initial reference style (M → Male styles, F → Female styles)
Target Voice — The uploaded audio file to clone
Reference Style Auto Auto finds the closest built-in style. None starts from random. Or pick a specific style.
Seed 49 Random seed for reproducibility
Speed 1.05 Speech speed multiplier during training (1.0 = natural speed)
Training Steps 3000 Maximum number of optimization steps
Learning Rate 0.0002 Optimizer learning rate (lower = more stable, slower)
Vocoder Steps 6 Number of denoising steps in the vocoder during evaluation
Save Every 500 Save a checkpoint every N optimization steps
Early Stop 0.015 Stop training when loss drops below this threshold
⚠️ GPU Recommended: Training uses PyTorch and is significantly faster with a CUDA-capable GPU. CPU training works but may take 30+ minutes.

🔊 Generate Speech

The Generate section lets you synthesize new speech using any trained or built-in voice style.

How to generate:

  1. Enter the text you want to synthesize
  2. Select the Language matching your text (31 languages supported)
  3. Pick a Voice Style — built-in styles are marked with 🔹, trained styles with ✅
  4. Adjust Speed and Vocoder Steps if desired
  5. Click Generate and wait for the result
💡 Tips:
  • The first generation takes longer because TTS models need to load into memory
  • Higher vocoder steps (8–10) produce better quality but take longer
  • Speed 1.0 = natural pace. Use 0.8–0.9 for slower, deliberate speech
  • Generated audio is automatically saved to the Samples section

📁 Samples

The Samples section displays all generated audio files. Files are sorted newest first.

  • Play any sample using the built-in audio player
  • Delete samples you no longer need with the trash icon
  • Sample filenames include the voice style name and a Unix timestamp
  • Files are stored in the samples/ directory

💡 Tips & Best Practices

Audio quality:

  • Use clean recordings with minimal background noise
  • Avoid music, multiple speakers, or heavy reverb
  • 44.1 kHz mono WAV is the ideal format
  • 10–60 seconds of continuous speech works best

Training:

  • Start with default settings — they work well in most cases
  • Watch the loss in the training log: lower = better speaker match
  • If loss plateaus high (> 0.1), try a different seed or reference style
  • Use checkpoints (e.g., _1000.json, _2000.json) to compare quality at different stages
  • Make sure Gender matches the target speaker

Troubleshooting:

  • Output sounds wrong gender — Check Gender setting matches the target speaker
  • Training too slow — Use a CUDA GPU if available. Reduce steps to 1500 for quick tests
  • Generation fails — Ensure the style JSON file exists and is valid
  • Audio artifacts — Try increasing vocoder steps to 8–10

CLI alternative:

python train_style.py --name my-voice --gender M --target-wav-path voices/my-voice.wav --num-steps 3000

python generate.py --text "Hello world" --style logs/my-voice/my-voice.json
